## 🚀 Deployment & Migration Strategy
1. **Backend (VPS/GlobeHost)**:
   - Host the Laravel application as a headless API.
   - Maintain the database and all media storage on this server.
   - Configure Nginx (`ntt_nginx.conf`) to serve the API and serve as a fallback.

2. **Frontend (Vercel)**:
   - Host the Next.js application.
   - Port the "Premium" design from Laravel Blade (`home_v1`, `post_v1`, etc.) to React components.
   - Connect to the VPS backend via API endpoints.

3. **Staging Environment**:
   - URL: `staging.newsthetruth.com` (Subdomain on VPS).
   - Purpose: API development and Backend verification.

## 🔄 Synchronization Protocol (For New Articles)
1. **Incremental Export**: Before cutover, focused export of: `posts`, `post_categories`, `post_metas`, `medias`.
2. **Media Sync**: Sync new files from `public/uploads/` on the live server.
3. **Merge Strategy**: Use a staging DB to verify IDs and slugs before the final production merge.
